# EZ-TuneIn Radio

Minimalist internet radio player. Plays Icecast/Shoutcast streams (SomaFM, SwissGroove, …)
and saves the current track to a CSV file at the click of a button.

- **Playback:** `just_audio` (+ `just_audio_media_kit` / libmpv on desktop)
- **Track metadata:** a small built-in ICY reader (`Icy-MetaData: 1` → `StreamTitle`),
  which works identically on Windows, Linux, and Android.
- **Saved file:** `radio_saved_tracks.csv` in the app's documents directory
  (columns: `timestamp,station,artist,title,album,raw`). Album is usually blank —
  ICY streams almost never carry it.

## What's scaffolded

This repo currently contains only the cross-platform source:

- `pubspec.yaml`
- `lib/main.dart`

The platform folders (`linux/`, `windows/`, `android/`) are generated by Flutter —
see step 2 below.

## Setup (Ubuntu — this machine)

1. **Install Flutter** (if not already). Easiest:
   ```
   sudo snap install flutter --classic
   flutter --version      # first run downloads the Dart SDK
   ```
   (Or follow https://docs.flutter.dev/get-started/install/linux for the tarball method.)

2. **Generate the platform scaffolding** in this directory. This adds `linux/`,
   `windows/`, `android/` without overwriting `lib/main.dart` or `pubspec.yaml`:
   ```
   flutter create --platforms=linux,windows,android .
   flutter pub get
   ```

3. **Linux desktop needs libmpv** (the `just_audio_media_kit` backend wraps it):
   ```
   sudo apt install -y libmpv-dev mpv
   ```

4. **Run it:**
   ```
   flutter run -d linux
   ```

## Building for Windows 11 (your primary target)

On a Windows machine with Flutter installed (`media_kit` bundles its own libs there,
no libmpv install needed):
```
flutter create --platforms=windows .
flutter pub get
flutter run -d windows         # or: flutter build windows
```

## Adding / changing stations

Use the **+** button in the app to add a station (name + direct stream URL), and
hover a station to reveal a delete button. Your list is saved between launches.
The defaults in `_defaultStations` (top of `lib/main.dart`) only seed the very
first launch.

> Stream URLs must be **direct** — not `.pls`/`.m3u` playlist links.
