# Screenshots

These images are shown in the top-level [`README.md`](../../README.md). Drop the
PNGs here with these **exact filenames** (the README links them by name):

| File | View | What to show |
|------|------|--------------|
| `player.png` | Player + now-playing | A station **playing**, with a live artist/title in the now-playing box and the station list visible. Pick a station with good metadata (e.g. SwissGroove or a SomaFM channel) so the track line is populated — that's the shot that sells the app. |
| `recording-settings.png` | Recording settings | Settings → Recording, scrolled so the **buffer-size slider and its guide caption** are visible (the caption showing the MB cost per minute at 128/256/320 kbps and how many minutes the chosen size rewinds). |
| `recordings-library.png` | Recordings library | The recordings list with at least one recorded song, ideally one **playing** so the seek bar / controls show. |

## Tips for good shots

- **Dark theme, real content.** The app is dark by default; a populated screen
  (real now-playing text, a couple of recorded songs) looks far better than an
  empty first-run window.
- **Trim the window chrome** if you can — capture just the app window, not the
  whole desktop.
- **Keep them a similar size/aspect** so the three sit evenly side by side. The
  README scales each to ~⅓ width; anything from ~800–1200 px wide is fine.
- **PNG**, reasonably compressed. These live in git, so avoid multi-MB files —
  a few hundred KB each is plenty.

Once the three files are here, the README section renders automatically (no
further edits needed).
